# Part 2 — 핵심 단어: UI/UX 디자이너 필수 어휘

> 단어 리스트 100개 외워봤자 입에서 안 나온다. 실무 문장 속에서 익혀야 쓸 수 있다.

---

## Chapter 5. 디자인 용어 영어로 말하기

디자인 리뷰에서 "이 부분이 좀... 어떻게 말하지..." 하다가 타이밍 놓친 적 있지? 아래 용어들을 **문장째로** 익혀라. 용어만 외우면 막상 입에서 안 나온다.

---

### 📌 레이아웃 & 구조 — 이걸 모르면 리뷰를 못 한다

| 용어 | 바로 쓰는 문장 |
|------|--------------|
| **layout** | The **layout** feels too cluttered. Let's simplify it. |
| **hierarchy** | The visual **hierarchy** isn't clear — the CTA gets lost. |
| **alignment** | Check the **alignment** — this element is off by 4px. |
| **spacing** | The **spacing** between components is inconsistent. |
| **padding** | Add more **padding** inside the card. |
| **whitespace** | More **whitespace** will make this feel less overwhelming. |
| **grid** | Everything should snap to the **grid** for consistency. |
| **breakpoint** | This layout breaks at the tablet **breakpoint**. |

💡 **가장 많이 쓰는 3개:** hierarchy, spacing, alignment. 이 3개만 자연스럽게 나와도 리뷰가 달라진다.

---

### 📌 타이포그래피 — 개발자한테 스펙 전달할 때 필수

| 용어 | 바로 쓰는 문장 |
|------|--------------|
| **font weight** | Use **font weight** 600 for headings. |
| **font size** | The body **font size** should be at least 16px. |
| **line height** | Increase the **line height** for better readability. |
| **letter spacing** | Reduce the **letter spacing** on the heading. |
| **readability** | The contrast ratio affects **readability**. |
| **legibility** | This font lacks **legibility** at small sizes. |
| **truncation** | Handle **truncation** gracefully with an ellipsis. |

⚠️ **readability vs legibility 헷갈리지 마라:**
- **readability** = 글 전체를 편하게 읽을 수 있는가 (줄 간격, 컬럼 너비 등)
- **legibility** = 글자 자체를 식별할 수 있는가 (폰트 선택, 크기)

---

### 📌 색상 & 스타일

| 용어 | 바로 쓰는 문장 |
|------|--------------|
| **contrast ratio** | The **contrast ratio** must meet WCAG AA standards. |
| **color token** | Update the **color token** in the design system. |
| **opacity** | Set the overlay **opacity** to 60%. |
| **dark mode** | Make sure all components support **dark mode**. |
| **primary color** | The **primary color** is used for CTAs and key actions. |

---

### 📌 컴포넌트 & 패턴 — 디자인 시스템 협업할 때

| 용어 | 바로 쓰는 문장 |
|------|--------------|
| **component** | This **component** should be reusable across screens. |
| **variant** | Create a disabled **variant** for this button. |
| **instance** | Update the main component — all **instances** will follow. |
| **design system** | This deviates from our **design system** guidelines. |
| **token** | Use spacing **tokens** instead of hard-coded values. |
| **affordance** | This button lacks **affordance** — it doesn't look clickable. |
| **state** | Define all **states**: default, hover, active, disabled, error. |
| **pattern** | We should follow the existing design **pattern** here. |

---

### 📌 UX 프로세스 용어 — 미팅에서 자연스럽게 써라

| 용어 | 바로 쓰는 문장 |
|------|--------------|
| **user flow** | Walk me through the **user flow** for onboarding. |
| **wireframe** | I'll start with low-fi **wireframes** before going high-fi. |
| **prototype** | Let's test this **prototype** with real users. |
| **handoff** | The **handoff** specs are in the Figma file. |
| **iteration** | This is the third **iteration** based on user feedback. |
| **edge case** | Don't forget to design for **edge cases**. |
| **pain point** | The biggest **pain point** is the onboarding length. |
| **friction** | Remove unnecessary **friction** from the checkout flow. |
| **mental model** | This goes against the user's **mental model**. |
| **accessibility** | **Accessibility** must be considered from the start. |

---

### ✏️ 바로 써봐 — 빈칸 채우기

1. 이 버튼은 클릭할 수 있게 생기지 않았어. → "This button lacks visual ______." `affordance`
2. 여백을 더 줘서 숨통 좀 틔우자. → "More ______ will make this feel less overwhelming." `whitespace`
3. 디자인 시스템에서 벗어났어. → "This deviates from our ______ guidelines." `design system`
4. 이 플로우에서 불필요한 마찰을 줄여야 해. → "Remove unnecessary ______ from this flow." `friction`

## Chapter 6. 비즈니스 필수 동사 100개

## Chapter 7. 뉘앙스를 바꾸는 형용사/부사

같은 피드백도 어떤 단어를 쓰느냐에 따라 받아들이는 사람의 반응이 완전히 달라진다. 이걸 모르면 의도치 않게 강하게 들리거나, 반대로 너무 흐릿하게 들린다.

---

### 📌 디자인 칭찬할 때 — 이 단어들로 구체적으로 칭찬해라

| 형용사 | 뜻 | 예문 |
|--------|-----|------|
| **intuitive** | 직관적인 | The flow feels really **intuitive**. |
| **seamless** | 매끄러운 | The transition is **seamless**. |
| **polished** | 완성도 높은 | This looks really **polished**. |
| **clean** | 깔끔한 | I love how **clean** the layout is. |
| **cohesive** | 통일감 있는 | The visual language feels more **cohesive** now. |
| **elegant** | 우아한 | Simple yet **elegant** — great job. |
| **scalable** | 확장 가능한 | The system is **scalable** for future needs. |

---

### 📌 문제 지적할 때 — 너무 강하지 않게

| 형용사 | 뜻 | 예문 |
|--------|-----|------|
| **cluttered** | 복잡한 | The screen feels a bit **cluttered**. |
| **confusing** | 혼란스러운 | This flow is **confusing** for new users. |
| **inconsistent** | 일관성 없는 | The spacing is **inconsistent** across screens. |
| **redundant** | 중복된 | This step seems **redundant** — can we remove it? |
| **overwhelming** | 너무 많은 | The homepage feels **overwhelming**. |
| **ambiguous** | 애매한 | The label is **ambiguous** — users might misread it. |
| **rigid** | 경직된 | The layout is too **rigid** for dynamic content. |

---

### ⚡ 이게 핵심 — 부사로 강도 조절하기

같은 지적도 부사 하나로 온도가 달라진다:

| 표현 | 강도 | 느낌 |
|------|------|------|
| "This doesn't work." | 🔥🔥🔥🔥🔥 | 직접적, 강함 |
| "This **significantly** affects usability." | 🔥🔥🔥🔥 | 객관적, 데이터 근거 |
| "This is **fairly** confusing." | 🔥🔥🔥 | 중립적 |
| "This is **slightly** off — worth revisiting." | 🔥🔥 | 부드럽고 열린 태도 |
| "This **might** be **a bit** confusing." | 🔥 | 가장 조심스러운 제안 |

---

### 📌 자주 쓰는 부사 — 상황별로 골라 써라

**강도 조절**

| 부사 | 뜻 | 예문 |
|------|-----|------|
| **slightly** | 약간 | The button is **slightly** too small for mobile. |
| **a bit** | 조금 | This feels **a bit** off. |
| **significantly** | 상당히 | This **significantly** reduces friction. |
| **overly** | 지나치게 | The instructions are **overly** detailed. |
| **barely** | 겨우 | The text is **barely** readable at this size. |

**의견 표현**

| 부사 | 뜻 | 예문 |
|------|-----|------|
| **honestly** | 솔직히 | **Honestly**, I don't think this direction is working. |
| **personally** | 개인적으로 | **Personally**, I prefer the second option. |
| **technically** | 기술적으로 | **Technically**, this is possible but it'll take time. |
| **visually** | 시각적으로 | **Visually**, the design feels heavy. |
| **ideally** | 이상적으로는 | **Ideally**, we'd test this before launch. |
| **realistically** | 현실적으로 | **Realistically**, we can only build one of these. |
| **ultimately** | 궁극적으로 | **Ultimately**, it's the user's experience that matters. |

---

### ✏️ 바로 써봐 — 부드럽게 바꾸기 연습

너무 직접적인 말을 부드럽게 바꿔봐:

1. "This is confusing." → `This is slightly confusing — worth revisiting.`
2. "The spacing is wrong." → `The spacing is a bit inconsistent here.`
3. "This won't work." → `I'm not sure this will work — could we explore other options?`
4. "I don't like this direction." → `Personally, I feel like this direction might not resonate with users.`
