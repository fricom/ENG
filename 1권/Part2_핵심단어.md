# 1권 Part 2 — 핵심 단어: UI/UX 디자이너 필수 어휘

## Chapter 5. 디자인 용어 영어로 말하기

> 피그마 코멘트, 디자인 리뷰, 개발 핸드오프에서 매일 쓰는 디자인 전문 용어

### 📌 레이아웃 & 구조

| 영어 용어 | 한국어 | 실무 예문 |
|-----------|--------|-----------|
| layout | 레이아웃 | The **layout** feels too cluttered. Let's simplify it. |
| hierarchy | 위계, 계층 | The visual **hierarchy** isn't clear — the CTA gets lost. |
| grid | 그리드 | Everything should snap to the **grid** for consistency. |
| alignment | 정렬 | Check the **alignment** — this element is off by 4px. |
| spacing | 간격 | The **spacing** between components is inconsistent. |
| padding | 패딩 (안쪽 여백) | Add more **padding** inside the card. |
| margin | 마진 (바깥 여백) | The **margin** between sections should be 48px. |
| whitespace | 여백 | More **whitespace** will make this feel less overwhelming. |
| breakpoint | 반응형 기준점 | This layout breaks at the tablet **breakpoint**. |
| viewport | 뷰포트 (화면 영역) | Test this on different **viewport** sizes. |

---

### 📌 타이포그래피

| 영어 용어 | 한국어 | 실무 예문 |
|-----------|--------|-----------|
| typeface / font | 서체 / 폰트 | We're using Inter as our primary **typeface**. |
| font weight | 폰트 굵기 | Use **font weight** 600 for headings. |
| font size | 폰트 크기 | The body **font size** should be at least 16px. |
| line height | 줄 간격 | Increase the **line height** for better readability. |
| letter spacing | 자간 | Reduce the **letter spacing** on the heading. |
| hierarchy (type) | 타입 위계 | We need a clearer type **hierarchy** — H1, H2, body. |
| readability | 가독성 | The contrast ratio affects **readability**. |
| legibility | 식별성 | This font lacks **legibility** at small sizes. |
| truncation | 말줄임 | Handle **truncation** gracefully with an ellipsis. |

---

### 📌 색상 & 스타일

| 영어 용어 | 한국어 | 실무 예문 |
|-----------|--------|-----------|
| color palette | 컬러 팔레트 | Let's define the **color palette** for the dark mode. |
| primary color | 주 색상 | The **primary color** is used for CTAs and key actions. |
| secondary color | 보조 색상 | Use the **secondary color** for supporting elements. |
| neutral | 중립색 | Backgrounds should use **neutral** tones. |
| contrast ratio | 대비율 | The **contrast ratio** must meet WCAG AA standards. |
| opacity | 투명도 | Set the overlay **opacity** to 60%. |
| gradient | 그라디언트 | Avoid **gradients** unless intentional. |
| color token | 컬러 토큰 | Update the **color token** in the design system. |
| dark mode | 다크 모드 | Make sure all components support **dark mode**. |

---

### 📌 컴포넌트 & 패턴

| 영어 용어 | 한국어 | 실무 예문 |
|-----------|--------|-----------|
| component | 컴포넌트 | This **component** should be reusable across screens. |
| variant | 변형 | Create a disabled **variant** for this button. |
| instance | 인스턴스 | Update the main component — all **instances** will follow. |
| pattern | 패턴 | We should follow the existing design **pattern** here. |
| design system | 디자인 시스템 | This deviates from our **design system** guidelines. |
| atomic design | 아토믹 디자인 | Our system follows **atomic design** principles. |
| token | 토큰 | Use spacing **tokens** instead of hard-coded values. |
| affordance | 행동 유도성 | This button lacks visual **affordance** — it doesn't look clickable. |
| feedback (UI) | 피드백 (반응) | Add visual **feedback** when the user clicks this. |
| state | 상태 | Define all **states**: default, hover, active, disabled, error. |

| 영어 용어 | 한국어 | 실무 예문 |
|-----------|--------|-----------|
| user flow | 사용자 플로우 | Walk me through the **user flow** for onboarding. |
| wireframe | 와이어프레임 | I'll start with low-fi **wireframes** before going high-fi. |
| prototype | 프로토타입 | Let's test this **prototype** with real users. |
| mockup | 목업 | I'll send you the final **mockup** by EOD. |

| edge case | 엣지 케이스 | Don't forget to design for **edge cases**. |
| pain point | 불편함 | The biggest **pain point** is the onboarding length. |

| accessibility | 접근성 | **Accessibility** must be considered from the start. |
| responsive design | 반응형 디자인 | Is this **responsive** for mobile? |
| information architecture | 정보 구조 | Let's revisit the **information architecture** of this section. |
| mental model | 멘탈 모델 | This goes against the user's **mental model**. |
| friction | 마찰, 불편함 | Remove unnecessary **friction** from the checkout flow. |

## Chapter 7. 뉘앙스를 바꾸는 형용사/부사

> 같은 문장도 어떤 형용사/부사를 쓰느냐에 따라 전달력이 완전히 달라진다.

### 📌 디자인 품질 형용사

| 형용사 | 뜻 | 예문 |
|--------|-----|------|
| intuitive | 직관적인 | The flow should feel **intuitive** to first-time users. |
| seamless | 매끄러운 | The transition between screens is **seamless**. |
| consistent | 일관성 있는 | The design needs to be more **consistent** across screens. |
| polished | 완성도 높은 | The final version looks really **polished**. |
| clean | 깔끔한 | I like how **clean** the layout is. |
| minimal | 미니멀한 | Go for a more **minimal** approach here. |
| elegant | 우아한 | The interaction is simple yet **elegant**. |
| responsive | 반응형인 | Make sure it's fully **responsive** on mobile. |
| accessible | 접근 가능한 | The design must be **accessible** to all users. |
| scalable | 확장 가능한 | We need a **scalable** component system. |
| cohesive | 통일감 있는 | The visual language feels more **cohesive** now. |
| robust | 견고한 | The design system is **robust** enough to handle edge cases. |

---

### 📌 문제를 지적하는 형용사

| 형용사 | 뜻 | 예문 |
|--------|-----|------|
| cluttered | 복잡한, 지저분한 | The screen feels too **cluttered**. |
| confusing | 혼란스러운 | This flow is **confusing** for new users. |
| inconsistent | 일관성 없는 | The spacing is **inconsistent** across screens. |
| redundant | 중복된, 불필요한 | This step seems **redundant** — can we remove it? |
| unclear | 불분명한 | The CTA is **unclear** — users don't know what will happen. |
| overwhelming | 압도적인, 너무 많은 | The homepage feels **overwhelming** with too much content. |
| outdated | 구식인 | The visual style feels **outdated**. |
| rigid | 경직된, 유연하지 않은 | The layout is too **rigid** for dynamic content. |
| fragmented | 파편화된 | The experience feels **fragmented** across platforms. |
| ambiguous | 애매한 | The label is **ambiguous** — users might misinterpret it. |

### 📌 강도를 조절하는 부사

> 같은 말도 부사 하나로 강하게, 혹은 부드럽게 만들 수 있다.

| 부사 | 뜻 | 예문 |
|------|-----|------|
| slightly | 약간 | The button is **slightly** too small for mobile. |
| a bit | 조금 | This feels **a bit** off — not sure why. |
| significantly | 상당히 | The new design **significantly** reduces friction. |
| considerably | 꽤, 상당히 | The load time has improved **considerably**. |
| fairly | 꽤, 상당히 | This is **fairly** straightforward to implement. |
| rather | 꽤, 다소 | This is **rather** complex for a first-time user. |
| quite | 꽤 | The feedback was **quite** positive overall. |
| overly | 지나치게 | The instructions are **overly** detailed. |
| barely | 겨우, 거의 ~않다 | The text is **barely** readable at this size. |
| relatively | 상대적으로 | This is **relatively** simple to fix. |
| approximately | 대략 | It'll take **approximately** two sprints to complete. |
| ideally | 이상적으로는 | **Ideally**, we'd run a test before launch. |

### 📌 뉘앙스 비교: 부사로 달라지는 피드백 온도

같은 문제를 지적할 때:

| 표현 | 강도 | 느낌 |
|------|------|------|
| This doesn't work. | ●●●●● | 직접적, 강함 |
| This **significantly** affects usability. | ●●●●○ | 객관적, 데이터 근거 |
| This is **fairly** confusing for users. | ●●●○○ | 중립적 |
| This is **slightly** off — worth revisiting. | ●●○○○ | 부드럽고 열린 태도 |
| This **might** be **a bit** confusing. | ●○○○○ | 가장 조심스러운 제안 |
