# 1권 Part 3 — 스피킹: 자신 있게 말하기

> 완벽한 문법보다 자신감 있게 전달하는 것이 먼저다. 원어민처럼 말할 필요 없다 — 알아듣게 말하면 된다.

---

## Chapter 8. 발음과 억양 기초 — 원어민이 알아듣는 말 만들기

### 🔑 한국인이 가장 많이 틀리는 발음 6가지

---

### 📌 1. V vs B

> 한국어에는 V 발음이 없어서 B로 대체하는 경우가 많다. 실무에서 헷갈리면 치명적.

| 단어 | 잘못된 발음 | 올바른 발음 | 의미 차이 |
|------|------------|------------|-----------|
| version | bersion ❌ | **v**ersion ✅ | 버전 |
| validate | balidate ❌ | **v**alidate ✅ | 검증하다 |
| visual | bisual ❌ | **v**isual ✅ | 시각적인 |
| review | rebiew ❌ | re**v**iew ✅ | 리뷰 |
| value | balue ❌ | **v**alue ✅ | 가치 |

**💡 발음 팁:** V는 윗니를 아랫입술에 살짝 대고 바람을 내보내며 발음. B는 양 입술을 붙였다 떼며 발음.

---

### 📌 2. F vs P

| 단어 | 잘못된 발음 | 올바른 발음 |
|------|------------|------------|
| flow | plow ❌ | **f**low ✅ |
| feedback | peedback ❌ | **f**eedback ✅ |
| font | pont ❌ | **f**ont ✅ |
| feature | peature ❌ | **f**eature ✅ |
| framework | pramework ❌ | **f**ramework ✅ |

**💡 발음 팁:** F는 V와 같은 방법으로, 윗니를 아랫입술에 대고 바람만 내보냄 (소리 없이).

---

### 📌 3. R vs L

> 영어에서 R과 L은 완전히 다른 소리. 특히 디자인 용어에 많이 등장.

| R 단어 | L 단어 | 구분 중요한 이유 |
|--------|--------|----------------|
| **r**ight | **l**ight | "right side" vs "light color" |
| **r**ead | **l**ead | 읽다 vs 이끌다 |
| **r**eal | **l**eal | 실제의 vs (충성스러운) |
| **r**oad map | **l**oad | 로드맵 vs 불러오다 |

**💡 발음 팁:**
- **R:** 혀를 어디에도 닿지 않게 말아 올리며 발음
- **L:** 혀끝을 윗니 뒤 잇몸에 살짝 붙이며 발음

---

### 📌 4. TH 발음

> 영어의 TH는 두 가지. 둘 다 혀를 앞니 사이에 내밀고 발음.

| 종류 | 예시 단어 | 연습 문장 |
|------|-----------|-----------|
| 유성 TH (the, this, that) | **the** design, **this** layout, **that** version | **The** feedback on **this** design was helpful. |
| 무성 TH (think, through, three) | **think**, **through**, **three** | I **think** we should **think** **through** **this** more. |

**💡 발음 팁:** 혀를 앞니 사이에 약간 내밀고, 유성은 목을 울려서, 무성은 바람만 내보내며 발음.

---

### 📌 5. 단어 강세 (Word Stress)

> 영어는 강세 언어. 어느 음절에 힘을 주느냐가 완전히 다른 단어처럼 들리게 한다.

| 단어 | 강세 위치 | 발음 표기 |
|------|-----------|----------|
| design | 뒤 | de-**SIGN** |
| feedback | 앞 | **FEED**-back |
| prototype | 앞 | **PRO**-to-type |
| component | 가운데 | com-**PO**-nent |
| hierarchy | 앞 | **HI**-er-ar-chy |
| usability | 가운데 | u-sa-**BI**-li-ty |
| accessibility | 가운데 | ac-ces-si-**BI**-li-ty |
| iteration | 가운데 | i-ter-**A**-tion |
| collaborate | 가운데 | col-**LAB**-o-rate |
| presentation | 뒤에서 두번째 | pres-en-**TA**-tion |

---

### 📌 6. 연음 (Linking Sounds)

> 원어민이 빠르게 말할 때 단어가 이어져 들리는 이유. 연음을 알면 듣기도 쉬워진다.

| 연음 패턴 | 예시 | 실제 발음 |
|-----------|------|-----------|
| 자음 + 모음 | **let us** → | "let-us" → "letus" |
| 같은 자음 반복 | **that time** → | "tha-time" |
| t/d 약화 | **what do** → | "wha-do" |
| 실무 표현 | "can I ask" → | "canya-sk" |
| 실무 표현 | "did you" → | "di-jou" |
| 실무 표현 | "what are you" → | "wha-tare-you" |

**자주 연음되는 실무 표현:**

| 원래 표현 | 실제 들리는 소리 |
|-----------|----------------|
| What do you think? | "Whadda-you think?" |
| I want to | "I wanna" |
| going to | "gonna" |
| Let me | "Lemme" |
| Give me | "Gimme" |
| kind of | "kinda" |
| a lot of | "a lotta" |

---

### 📌 억양 패턴 — 올라가면 질문, 내려가면 확신

| 패턴 | 언제 | 예시 |
|------|------|------|
| 내려가는 억양 ↘ | 확신, 결론, 정보 전달 | This is the final version. ↘ |
| 올라가는 억양 ↗ | Yes/No 질문 | Did you review it? ↗ |
| 올라갔다 내려오는 억양 ↗↘ | 목록 나열 | We need to fix the layout ↗, the colors ↗, and the spacing. ↘ |
| 강조 | 중요한 단어에 힘 주기 | This is **really** important. |

## Chapter 9. 말이 막힐 때 쓰는 표현 — 필러와 시간 벌기 전략

> 말이 막혔을 때 침묵하거나 "음..."만 반복하면 자신감 없어 보인다. 프로처럼 시간 버는 법을 익힌다.

### 🔑 필러 vs 시간 벌기 표현

| 종류 | 설명 | 예시 |
|------|------|------|
| 필러 (Filler) | 생각하는 동안 말 이어가기 | Well, Let me see, You know... |
| 시간 벌기 | 질문 받았을 때 답 생각하기 | That's a great question, Let me think... |
| 확인 요청 | 못 들었거나 이해 안 됐을 때 | Could you repeat that? |
| 재구성 | 내 이해가 맞는지 확인 | So what you're saying is...? |

---

### 📌 자연스러운 필러 표현

> "음...", "어..." 대신 쓸 수 있는 자연스러운 영어 필러

| 필러 | 뉘앙스 | 예문 |
|------|--------|------|
| Well, ... | 가장 범용적인 필러 | **Well**, I think we should reconsider the direction. |
| So, ... | 이어서 말할 때 | **So**, what I was thinking is... |
| Let me see... | 잠깐 생각할 때 | **Let me see**... I think the issue is the hierarchy. |
| You know, ... | 공감 구할 때 (격식 없는 자리) | **You know**, it's a tricky balance. |
| I mean, ... | 정정하거나 강조할 때 | **I mean**, it looks okay, but something feels off. |
| Actually, ... | 추가/수정 정보 줄 때 | **Actually**, I changed my mind on this. |
| Honestly, ... | 솔직한 의견 말할 때 | **Honestly**, I'm not sure this is the right approach. |
| Basically, ... | 핵심 요약할 때 | **Basically**, the problem is that users can't find the button. |

---

### 📌 시간 벌기 표현 — 질문 받았을 때

> 바로 대답 못할 때 쓰는 표현. 침묵보다 훨씬 프로답게 들린다.

**생각할 시간 벌기**

| 표현 | 상황 |
|------|------|
| That's a great question. | 가장 범용적, 어디서나 |
| Let me think about that for a second. | 잠깐 생각이 필요할 때 |
| That's something I need to think through. | 복잡한 질문 받았을 때 |
| I want to make sure I give you a good answer. | 신중하게 답하고 싶을 때 |
| Can I get back to you on that? | 지금 당장 답 모를 때 |
| I'll need to check and follow up. | 확인이 필요할 때 |
| Let me pull up the file real quick. | 파일 확인이 필요할 때 |

---

### 📌 모를 때 / 이해 못했을 때

> 모른다고 하는 것은 약점이 아니다. 어떻게 말하느냐가 중요.

**못 들었을 때**

| 표현 | 뉘앙스 |
|------|--------|
| Sorry, could you repeat that? | 가장 기본 |
| I'm sorry, I didn't catch that. | 정중한 표현 |
| Could you say that one more time? | 부드러운 표현 |
| Sorry, the connection cut out for a second — what was that? | 화상 회의에서 |

**이해가 안 됐을 때**

| 표현 | 뉘앙스 |
|------|--------|
| Could you clarify what you mean by that? | 의미가 모호할 때 |
| I want to make sure I understand — are you saying ~? | 내 이해 확인 |
| Could you elaborate a bit more? | 더 자세한 설명 요청 |
| Could you give me an example? | 추상적인 말을 구체화 |
| I'm not familiar with that term — could you explain? | 모르는 용어 나왔을 때 |

**모르는 내용 질문 받았을 때**

| 표현 | 뉘앙스 |
|------|--------|
| I'm not 100% sure, but I think... | 확신 없이 말할 때 |
| I don't have that information right now, but I'll find out. | 모를 때 솔직하게 |
| That's outside my area — let me connect you with the right person. | 전문 영역 벗어날 때 |
| I'd rather not guess — let me verify and get back to you. | 정확성을 위해 확인하겠다 |

---

### 📌 말 이어가기 / 화제 전환

| 표현 | 상황 |
|------|------|
| Going back to what I was saying... | 원래 하던 말로 돌아갈 때 |
| On a related note... | 관련된 주제로 넘어갈 때 |
| That said... / Having said that... | 앞 내용과 반대 의견 낼 때 |
| Anyway, ... | 본론으로 돌아올 때 |
| To get back on track... | 회의가 옆길로 샐 때 |
| Before we move on... | 다음 주제 넘어가기 전 확인 |
| Speaking of which... | 자연스럽게 관련 주제로 |

---

### 📌 의견이 다를 때 — 부드럽게 반박하기

> 직접 "No, you're wrong"은 피한다. 의견 차이를 표현하는 세련된 방법.

| 상황 | 표현 |
|------|------|
| 다른 관점 제시 | I see it a bit differently — from my perspective... |
| 부드러운 반대 | I understand where you're coming from, but I wonder if... |
| 추가 고려 사항 | That makes sense, though I'm a little concerned about... |
| 데이터로 반박 | The user research actually suggests something different. |
| 조심스러운 반대 | I might be wrong, but I feel like this could be an issue. |
| 확인 요청 | Have you considered the impact on mobile users? |

## Chapter 10. 프레젠테이션 목소리 만들기 — 강약, 속도, 끊어 읽기

> 발표는 내용만큼 전달 방식이 중요하다. 같은 내용도 어떻게 말하느냐에 따라 설득력이 완전히 달라진다.

### 🔑 프레젠테이션 목소리 4가지 요소

| 요소 | 핵심 원칙 |
|------|-----------|
| 속도 (Pace) | 평소보다 20% 천천히. 중요한 부분은 더 천천히. |
| 강조 (Stress) | 키워드에 힘을 준다. 문장 전체를 강조하면 아무것도 강조 안 된 것. |
| 쉬기 (Pause) | 침묵을 두려워하지 말 것. 쉼표가 청중이 생각할 시간을 준다. |
| 억양 (Intonation) | 단조롭게 말하지 않는다. 올리고 내리며 리듬을 만든다. |

---

### 📌 발표 시작 — 오프닝 표현

| 표현 | 상황 |
|------|------|
| Thank you for joining today. | 가장 무난한 시작 |
| Let me start by giving you a quick overview. | 전체 구조 예고 |
| Today I'm going to walk you through... | 내용 예고 |
| I'll keep this brief and leave time for questions. | 시간 관리 예고 |
| Before I dive in, does everyone have the Figma link? | 준비 확인 |
| I'll share my screen — let me know if you can see it. | 화면 공유 시작 |

---

### 📌 발표 구조 신호 — 청중이 따라오게 하기

> 지금 어디 있는지, 다음에 뭐가 오는지 알려주는 표현들

**도입**

| 표현 | 상황 |
|------|------|
| Let me start with a quick context. | 배경 설명 시작 |
| First, let me explain why we're here. | 목적 설명 |
| To give you some background... | 배경 정보 제공 |
| The problem we're trying to solve is... | 문제 정의 |

**본론 전환**

| 표현 | 상황 |
|------|------|
| Now let's look at the design. | 본론으로 넘어갈 때 |
| Moving on to the next screen... | 다음 화면으로 |
| This brings me to... | 자연스러운 전환 |
| Let me zoom in on this section. | 특정 부분 강조 |
| Here's where it gets interesting. | 핵심 포인트 예고 |
| What I want to highlight here is... | 강조 포인트 |

**마무리**

| 표현 | 상황 |
|------|------|
| To summarize... | 요약 시작 |
| The key takeaway here is... | 핵심 정리 |
| So to wrap up... | 마무리 시작 |
| That brings me to the end of my presentation. | 마무리 선언 |
| I'll open it up for questions now. | Q&A 시작 |
| What questions do you have? | Q&A 시작 (더 적극적) |

---

### 📌 강조하는 법 — 키워드에 힘 주기

> 문장 전체를 강하게 말하지 않는다. 핵심 단어 하나에만 힘을 준다.

| 원래 문장 | 강조 버전 | 의미 변화 |
|-----------|-----------|-----------|
| This is the main issue. | This is the **MAIN** issue. | 다른 게 아니라 이게 핵심 |
| Users couldn't find the button. | **USERS** couldn't find the button. | 개발자가 아니라 사용자가 |
| The drop-off rate increased by 30%. | The drop-off rate increased by **30%**. | 수치 강조 |
| We need to fix this before launch. | We need to fix this **before launch**. | 시점 강조 |

---

### 📌 효과적인 쉬기 (Pause)

| 언제 | 이유 | 예시 |
|------|------|------|
| 중요한 말 하기 직전 | 청중이 집중하게 만들기 | "The biggest issue is... (1초 쉬기) the navigation." |
| 숫자/데이터 말한 후 | 청중이 숫자를 받아들일 시간 | "Drop-off rate was 70%. (2초 쉬기) That's huge." |
| 질문 던진 후 | 청중이 생각하게 만들기 | "Why do users drop off here? (1초 쉬기) Let me show you." |
| 섹션 전환 시 | 새 섹션 시작 신호 | "That's the problem. (1초 쉬기) Now let's look at the solution." |

---

### 📌 Q&A 핸들링

**질문 받았을 때**

| 표현 | 상황 |
|------|------|
| Great question — let me address that. | 가장 무난한 시작 |
| I'm glad you brought that up. | 좋은 질문 받았을 때 |
| That's exactly what I was going to cover next. | 다음 내용과 연결 |
| Let me make sure I understand your question — are you asking about ~? | 질문 확인 |

**답을 모를 때**

| 표현 | 상황 |
|------|------|
| I don't have that data right now, but I can follow up. | 데이터 없을 때 |
| That's a good point — I'll need to check and get back to you. | 확인 필요할 때 |
| I want to give you an accurate answer, so let me verify that. | 정확성 강조 |
| That's outside the scope of today's presentation, but happy to discuss offline. | 범위 벗어날 때 |

**여러 사람 의견 조율할 때**

| 표현 | 상황 |
|------|------|
| That's a valid point. Does anyone else have thoughts on this? | 의견 확장 |
| I hear two different perspectives — let me summarize both. | 의견 정리 |
| Can we table this for now and follow up offline? | 시간 부족할 때 |
| Let's take this offline so we don't run over time. | 오프라인으로 넘기기 |
