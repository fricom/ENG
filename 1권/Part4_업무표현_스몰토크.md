## Chapter 13. 의견 말하기, 동의/반대하기, 제안하기

### 📌 의견 말하기

**확신 있을 때**

| 표현 | 강도 |
|------|------|
| I think we should go with option A. | ●●●○○ |
| I strongly believe this is the right direction. | ●●●●○ |
| I'm confident that this approach will work. | ●●●●○ |
| This is clearly the better choice. | ●●●●● |
| In my opinion, the second option is stronger. | ●●●○○ |

**확신 없을 때**

| 표현 | 강도 |
|------|------|
| I might be wrong, but I feel like... | ●○○○○ |
| I'm not 100% sure, but my instinct says... | ●●○○○ |
| I could be off here, but it seems like... | ●○○○○ |
| This is just a gut feeling, but... | ●●○○○ |
| I'm on the fence about this, to be honest. | 중립 |

---

### 📌 동의하기

**완전 동의**

| 표현 | 뉘앙스 |
|------|--------|
| Exactly. | 강한 동의 |
| Absolutely. | 강한 동의 |
| That's a great point. | 상대 의견 인정 |
| I couldn't agree more. | 완전 동의 |
| You're totally right. | 캐주얼한 동의 |
| That's exactly what I was thinking. | 같은 생각이었다 |

**부분 동의**

| 표현 | 뉘앙스 |
|------|--------|
| I agree with you on that, and I'd also add... | 동의 + 추가 |
| That makes sense — though I wonder if... | 동의 + 의문 |
| I see your point, and I think that's largely true, but... | 조건부 동의 |
| You raise a good point — I just want to make sure we also consider... | 다른 관점 추가 |

---

### 📌 반대하기 — 부드럽게, 그러나 명확하게

> "No" 대신 쓸 수 있는 표현들. 관계를 유지하면서 의견을 전달하는 것이 핵심.

**부드러운 반대**

| 표현 | 뉘앙스 |
|------|--------|
| I see it a bit differently. | 다른 관점 |
| I'm not sure I fully agree. | 조심스러운 반대 |
| I have some concerns about that direction. | 우려 표현 |
| I'm not totally convinced — can we explore other options? | 열린 반대 |
| That's one way to look at it, but... | 다른 시각 제시 |

**데이터/근거 기반 반대**

| 표현 | 뉘앙스 |
|------|--------|
| The user research actually points in a different direction. | 데이터 근거 |
| Based on what we saw in testing, I'm not sure this will work. | 테스트 근거 |
| I'd want to validate that assumption before committing. | 검증 요청 |
| The data suggests users prefer a different approach. | 데이터 인용 |

**팀 관점으로 반대**

| 표현 | 뉘앙스 |
|------|--------|
| I want to make sure we're not moving too fast on this. | 속도 우려 |
| Can we pressure-test this a bit more before deciding? | 더 검토 요청 |
| I think we should consider the long-term implications. | 장기적 관점 |
| Have we thought about how this affects the other flows? | 영향도 고려 요청 |

## Chapter 14. 회의 유형별 표현 — 1:1 싱크, 킥오프, 회고(레트로)

> 매니저나 팀원과 정기적으로 갖는 미팅. 업무 현황, 블로커, 성장 이야기를 나눈다.

**시작**

| 표현 | 상황 |
|------|------|
| How are you doing this week? | 근황 물어보기 |
| Anything pressing you want to cover first? | 우선순위 확인 |
| I have a few things to go over — is there anything on your end first? | 아젠다 조율 |

**진행 중**

| 표현 | 상황 |
|------|------|
| I'm currently working on X, and I'm a bit blocked on Y. | 현황 공유 |
| I wanted to flag a concern I've been thinking about. | 우려 공유 |
| I'd love your feedback on how I handled this situation. | 피드백 요청 |
| Is there anything you think I should be doing differently? | 개선점 요청 |
| I wanted to talk about my career growth — do you have time for that? | 성장 대화 요청 |
| I'm feeling a bit overwhelmed with the current workload. | 솔직한 상황 공유 |
| I've been thinking about taking on more responsibility. | 성장 의지 표현 |

**마무리**

| 표현 | 상황 |
|------|------|

### 📌 킥오프 미팅 (Kickoff)

> 새 프로젝트의 시작을 알리는 회의. 목표, 범위, 역할, 일정을 정렬한다.

**프로젝트 소개**

| 표현 | 상황 |
|------|------|
| Let me give a quick overview of what we're building. | 프로젝트 소개 |
| The goal of this project is to... | 목표 명시 |
| The problem we're solving is... | 문제 정의 |
| Our target users are... | 대상 사용자 |
| This project is being driven by... | 배경 설명 |

**범위 & 역할**

| 표현 | 상황 |
|------|------|
| Here's what's in scope and what's out of scope. | 범위 명확화 |
| Design will own X, and engineering will own Y. | 역할 분담 |
| Who will be the decision maker on the design side? | 결정권자 확인 |
| I'll be the main point of contact for design. | 내 역할 소개 |
| Let's agree on how we'll communicate — Slack or email? | 커뮤니케이션 채널 합의 |

**일정 & 다음 단계**
